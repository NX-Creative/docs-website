---
title: Exports
description: Public exports registered by nx_fourseasons. Client-side weather state plus the server-side Weather Platform API.
---

`nx_fourseasons` registers client exports for weather state, sync, road
traction, and screen FX, plus server exports that form the Weather
Platform API. Call client exports from any client script and server
exports from any server script, both via `exports.nx_fourseasons:<Name>()`.

The server exports are gated by `Config.API.Enabled`. The weather-control
exports (`RequestWeather`, `LockWeather`, `ReleaseWeather`,
`GetWeatherControlOwner`) also require `Config.API.AllowControl`.

## State (client)

<ExportSignature
  name="exports.nx_fourseasons:GetCurrentWeather"
  side="client"
  description="Return the current weather snapshot for this client."
  returns="{ weather: string, type: string, intensity: number, temperature: number, season: string, storm: table | nil }"
>
```lua
local w = exports.nx_fourseasons:GetCurrentWeather()
print(w.weather, w.intensity, w.temperature)
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:GetCurrentTemperature"
  side="client"
  description="Return the current temperature in Celsius."
  returns="number"
>
```lua
local temp = exports.nx_fourseasons:GetCurrentTemperature()
if temp < 0 then
    -- Player is in sub-zero conditions
end
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:GetCurrentSeason"
  side="client"
  description="Return the active season id."
  returns="'SPRING' | 'SUMMER' | 'FALL' | 'WINTER'"
>
```lua
local season = exports.nx_fourseasons:GetCurrentSeason()
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:GetCurrentTime"
  side="client"
  description="Return the synced clock for this client."
  returns="{ hour: number, minute: number, second: number, day: number, month: number, year: number, useRealTime: boolean, freeze: boolean }"
>
```lua
local t = exports.nx_fourseasons:GetCurrentTime()

```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:GetCurrentStormState"


</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:IsPlayerIndoors"

## Exposure (client)

<ExportSignature
  name="exports.nx_fourseasons:GetExposureLevel"
  side="client"
  description="Return the local player's current body-temperature exposure. level is 0 (comfortable) to 3 (severe). kind is the direction of the danger, or false when comfortable. Available only while Config.Exposure.Enabled is true."
  returns="{ level: number, kind: 'cold' | 'heat' | false }"
>
```lua
local e = exports.nx_fourseasons:GetExposureLevel()
if e.level >= 2 and e.kind == 'cold' then
    -- Player is in cold danger
end
```
</ExportSignature>

## Forecast (client)

<ExportSignature
  name="exports.nx_fourseasons:GetForecast"
  side="client"
  description="Return the server-provided forecast. With no argument, returns the full forecast array. With a day offset, returns that single day."
  params={[
    { name: 'daysAhead', type: 'number', optional: true, description: '0 for today, 1 for tomorrow, up to Config.Forecast.Days - 1.' }
  ]}
  returns="table[] | { weather: string, type: string, temperature: number, intensity: number, day?: number, date?: any }"
>
```lua
-- Full forecast
local days = exports.nx_fourseasons:GetForecast()

-- Single day
local tomorrow = exports.nx_fourseasons:GetForecast(1)
print(tomorrow.weather, tomorrow.temperature)
```
</ExportSignature>

## Sync control (client)

Pause or resume the weather and time sync for the local player. Useful
for cutscenes, interiors, and minigames that run in an isolated
environment.

<ExportSignature
  name="exports.nx_fourseasons:PauseSync"
  side="client"
  description="Pause or resume weather and time sync on this client. Pending updates are replayed on resume."
  params={[
    { name: 'paused', type: 'boolean', description: 'true to pause, false to resume.' }
  ]}
  returns="boolean"
>
```lua
-- Freeze weather during a cutscene
exports.nx_fourseasons:PauseSync(true)

-- Resume after
exports.nx_fourseasons:PauseSync(false)
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:RequestSync"
  side="client"
  description="Ask the server to resend the full weather, forecast, and storm state to this client."
  returns="true"
>
```lua
exports.nx_fourseasons:RequestSync()
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:IsSyncPaused"
  side="client"
  description="Return whether sync is currently paused on this client."
  returns="boolean"
>
```lua
if exports.nx_fourseasons:IsSyncPaused() then
    exports.nx_fourseasons:PauseSync(false)
end
```
</ExportSignature>

## Road traction (client)

Override traction reductions on individual vehicles. Useful for scripted
missions where a vehicle should ignore the active weather, or for
adding a stronger penalty to a specific vehicle class.

<ExportSignature
  name="exports.nx_fourseasons:ToggleCurrentRoadEffects"
  side="client"
  description="Enable or disable the active weather-driven road effect on a vehicle."
  params={[
    { name: 'vehicle', type: 'number', description: 'Vehicle entity handle.' },
    { name: 'enabled', type: 'boolean', description: 'true to apply, false to suspend.' }
  ]}
>
```lua
local veh = GetVehiclePedIsIn(PlayerPedId(), false)
exports.nx_fourseasons:ToggleCurrentRoadEffects(veh, false)
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:SetVehicleRoadEffect"
  side="client"
  description="Override the traction reduction applied to a vehicle. Value replaces the active weather default."
  params={[
    { name: 'vehicle', type: 'number' },
    { name: 'overrideReduction', type: 'number', description: 'Traction reduction in the range 0.0 (none) to 1.0 (max).' }
  ]}
>
```lua
exports.nx_fourseasons:SetVehicleRoadEffect(veh, 0.5)
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:GetVehicleRoadEffect"
  side="client"
  description="Return the road-effect state currently tracked for a vehicle."
  params={[
    { name: 'vehicle', type: 'number' }
  ]}
  returns="table | nil"
/>

<ExportSignature
  name="exports.nx_fourseasons:ResetVehicleRoadEffect"
  side="client"
  description="Clear any override set by SetVehicleRoadEffect and reapply the active weather default."
  params={[
    { name: 'vehicle', type: 'number' }

## Weather control (server)

Request or lock the weather from another resource. Registered only when
both `Config.API.Enabled` and `Config.API.AllowControl` are true. Every
lock is attributed to the invoking resource and auto-reverts when that
resource stops. Higher priority wins; the applied owner is published on
`GlobalState.nxWeatherLockOwner`.

<ExportSignature
  name="exports.nx_fourseasons:RequestWeather"
  side="server"
  description="Request a weather type at default priority 10. Returns a token, or false if control is disabled or the caller is unattributable. Auto-expires after opts.durationMs when provided."
  params={[
    { name: 'weatherType', type: 'string', description: "A GTA weather type, e.g. 'THUNDER' or 'SNOW'." },
    { name: 'opts', type: 'table', optional: true, description: '{ intensity?: number (0.0-1.0), durationMs?: number, priority?: number }. Intensity is snapshotted at register time; omit to reuse the current value.' }
  ]}
  returns="string | false"
>
```lua
-- Ask for thunder for 10 minutes
local token = exports.nx_fourseasons:RequestWeather('THUNDER', { durationMs = 600000 })
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:LockWeather"
  side="server"
  description="Lock a weather type at default priority 100. A lock never auto-expires: any durationMs is ignored. Returns a token, or false if control is disabled or the caller is unattributable."
  params={[
    { name: 'weatherType', type: 'string' },
    { name: 'opts', type: 'table', optional: true, description: '{ intensity?: number (0.0-1.0), priority?: number }.' }
  ]}
  returns="string | false"
>
```lua
local token = exports.nx_fourseasons:LockWeather('EXTRASUNNY')
-- ... later
exports.nx_fourseasons:ReleaseWeather(token)
```
</ExportSignature>

<ExportSignature
  name="exports.nx_fourseasons:ReleaseWeather"
  side="server"
  description="Release the calling resource's lock. The token must match the lock owned by the invoking resource. A nil token releases the caller's own lock unconditionally. Returns true when a lock was released."
  params={[
    { name: 'token', type: 'string', optional: true, description: 'The token returned by RequestWeather or LockWeather.' }
  ]}
  returns="boolean"
/>

<ExportSignature
  name="exports.nx_fourseasons:GetWeatherControlOwner"
  side="server"
  description="Return the resource name whose lock is currently applied, or nil when no lock is active."
  returns="string | nil"
/>
