All player-facing settings live in `config.lua`. The file is in
`escrow_ignore` so it remains editable after install.

Locked values that drive the calibrated RTP (symbol weights, payout
tables, bonus weights, max-win cap, DUI panel geometry, sync radius)
live in `constants.lua` and are not part of this page. Editing them
will skew payout shape and is not supported.

## Currencies

Each slot cabinet runs on one currency. Admins assign it per cabinet
through the `/manageslots` menu, and one global option covers the
in-world Diamond Casino cabinets. Define every selectable currency in
`Config.Currencies`.

<ConfigOption name="Config.Currencies" type="table[]" default="{ { key = 'money', label = 'Cash', symbol = '$', type = 'item', item = 'money' }, { key = 'chips', label = 'Casino Chips', symbol = '◆', type = 'item', item = 'casino_chip' } }" description="List of currency options a slot can use. Each entry is charged on a bet and paid out on a win. See the field reference below." />

Each entry takes:

- `key`. Unique internal id stored in the database. Do not rename in place.
- `label`. Display label shown in the `/manageslots` menu.
- `symbol`. Short symbol drawn in the cabinet UI, for example `"$"`, `"◆"`, or `"C"`.
- `type`. Either `"account"` (framework money account) or `"item"` (ox_inventory item).
- `account`. Required when `type` is `"account"`. The framework account to charge and pay. ESX: `"money"`, `"bank"`, or `"black_money"`. QB/QBox: `"cash"`, `"bank"`, or `"crypto"`.
- `item`. Required when `type` is `"item"`. The ox_inventory item used as one chip.

The first entry is the default for newly placed slots and for the
in-world casino cabinets until an admin overrides them in-game.

<ConfigOption name="Config.DefaultCurrency" type="string" default="nil" description="Optional currency key that sets which Config.Currencies entry new slots are created with. When nil, the first entry is used. Lets you reorder the menu without changing the default." />

## Slot placement

The placement system powers the `/manageslots` admin command. It
controls who is allowed to open the manager, and optionally lets
specific jobs purchase and place slot cabinets.

<ConfigOption name="Config.SlotPlacement.TableName" type="string" default="'nx_slotmachine_world_slots'" description="Database table that stores placed slot cabinets. If you change this, update sql/install.sql to match." />

<ConfigOption name="Config.SlotPlacement.Command" type="string" default="'manageslots'" description="Chat command that opens the placement manager. Renaming this updates the registered command name." />

### Access

<ConfigOption name="Config.SlotPlacement.Access.Admins" type="table" default="{ Enabled=true, ESXGroups={'admin','superadmin'}, QBPermissions={'admin'}, QBoxPermissions={'admin'}, QBoxGroups={'admin'}, AcePermissions={'command.manageslots','nx_slotmachine.manageslots'} }" description="Who counts as an admin for slot placement. Any one matching group, permission, or ACE check grants access." />

<ConfigOption name="Config.SlotPlacement.Access.Jobs" type="table" default="{ Enabled=false, DefaultPrice=0, Allowed={} }" description="Optional job-based access.
