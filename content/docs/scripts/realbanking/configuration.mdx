All settings live in `shared/config.lua`. The file is in `escrow_ignore`
so it remains editable after install. Locale strings live in
`shared/locales/<lang>.lua`, and server-only secrets (Discord webhooks)
live in `server-config.lua`, which is not covered here.

## 3D NUI panels

Configures the per-ATM-model 3D rendering of the NUI via `nx-3d`. Only
customize if you add new ATM models. The shipped models
(`prop_fleeca_atm`, `prop_atm_01`, `prop_atm_02`) are already calibrated.

<ConfigOption name="Config.NUI3D.enabled" type="boolean" default="true" description="Master toggle for 3D panels. If false, UI renders as a flat overlay." />
<ConfigOption name="Config.NUI3D.debug" type="boolean" default="false" description="Render debug spheres / wireframes for panel positioning." />
<ConfigOption name="Config.NUI3D.resource" type="string" default="'nx-3d'" description="Resource name that provides the 3D rendering." />
<ConfigOption name="Config.NUI3D.resW" type="number" default="1024" description="DUI texture width." />
<ConfigOption name="Config.NUI3D.resH" type="number" default="1024" description="DUI texture height." />
<ConfigOption name="Config.NUI3D.interactDistance" type="number" default="2.5" description="Distance within which finger-IK interaction activates." />
<ConfigOption name="Config.NUI3D.updateInterval" type="number" default="50" description="Panel update interval in milliseconds." />
<ConfigOption name="Config.NUI3D.updateMaxDistance" type="number" default="5.0" description="Distance beyond which the panel stops updating entirely." />

<ConfigOption name="Config.NUI3D.fingerInteraction" type="table" description="IK finger-press configuration: enable flag, cursor hide, PED IK part / blend timings, per-axis screen-press response, per-ped height normalization (so taller peds / high heels don't misalign the hand). Values are tuned for mp_m_freemode_01." />

<ConfigOption name="Config.NUI3D.panels" type="table" description="Per-model panel calibration keyed by prop hash. Each entry defines screen / keypad dimensions, tilt, IK offsets, and the card-prop attachment pose. Shipped entries: prop_fleeca_atm, prop_atm_01, prop_atm_02. Recalibrate in-world using Config.NUI3D.debug = true." />
