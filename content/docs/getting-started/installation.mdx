Every NX Creative script assumes a baseline FiveM server with a supported
framework and a small set of ox-ecosystem resources in place. Get those
right once and individual scripts drop in without surprises.

## Install checklist

1. **Download** — grab the latest release `.zip` from your Tebex purchase.
2. **Unpack** — drop the folder into `resources/[nx]/` (or wherever you
   keep third-party resources).
3. **Import the SQL** — each script with persisted state ships a
   `sql/install.sql` file. Run it once.
4. **Edit `config.lua`** — per-script settings live here. Don't touch
   `shared.lua` or `server.lua` unless the script's docs say so.
5. **Add to `server.cfg`** — ensure the script starts _after_ its
   dependencies:

   ```ini
   ensure ox_lib
   ensure ox_target
   ensure ox_inventory
   ensure oxmysql
   ensure nx_<script>
   ```

6. **Boot and verify** — on first start, console prints:

   ```
   [nx] nx_<script> — running on <framework>
   ```

   If you see that line, you're good.

## Troubleshooting

If the script refuses to start, the answer is almost always one of:

- A dependency isn't running or is the wrong version.
- SQL wasn't imported or imported into the wrong database.
- The license key isn't bound to this server (check Tebex Keymaster).

See [Support](/docs/getting-started/support) if none of those apply.
